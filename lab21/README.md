# Лабораторна робота №21
## Тема: OCP: гнучкі алгоритми розрахунку (Factory/Strategy)
## Варіант 7: Хмарне сховище

### Мета роботи
Застосувати принцип відкритості/закритості (OCP) для створення гнучкої системи розрахунків за допомогою патернів Factory Method та Strategy, забезпечивши можливість легкого додавання нових алгоритмів без зміни існуючого коду.

## Завдання вар. 7 
Створити систему розрахунку вартості підписки на хмарне сховище:
**Параметри розрахунку**: обсяг даних (ГБ), кількість користувачів.
**Стратегії**:
PersonalPlan — тариф для особистого використання.
BusinessPlan — тариф для бізнесу.
EnterprisePlan — корпоративний тариф.
EducationalPlan — навчальний тариф (додаткова стратегія для демонстрації OCP).


## Принцип OCP
**Закриті для модифікації**: `CloudStorageService` працює з будь-якою стратегією через інтерфейс.
**Відкриті для розширення**: додавання `EducationalPlan` не потребувало змін в існуючих класах.

## Висновок
Створено гнучку систему розрахунку з використанням патернів **Strategy** та **Factory Method**. Дотримано принцип OCP — додавання нової стратегії не потребує зміни існуючого коду.
