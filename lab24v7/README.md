Лабораторна робота №24

Тема: Strategy + Observer: динамічна підстановка алгоритмів + тести

Мета роботи
Застосувати патерни Strategy та Observer для створення гнучкої системи, яка дозволяє динамічно змінювати алгоритми обробки даних та автоматично сповіщати залежні компоненти про зміни.

1. Патерн Strategy
Реалізовані компоненти:
INumericOperationStrategy — інтерфейс стратегії з методом Execute(double value)
SquareOperationStrategy — обчислення квадрату числа
CubeOperationStrategy — обчислення кубу числа
SquareRootOperationStrategy — обчислення квадратного кореня
NumericProcessor — контекстний клас, який використовує стратегію та дозволяє її змінювати через метод SetStrategy()
Принцип роботи Strategy:
Клас NumericProcessor делегує виконання операції поточній стратегії, що дозволяє змінювати алгоритм обробки даних без зміни коду самого процесора.

2. Патерн Observer
Реалізовані компоненти:
ResultPublisher — суб'єкт, який має подію ResultCalculated та метод PublishResult() для сповіщення підписників
ConsoleLoggerObserver — виводить результати в консоль
HistoryLoggerObserver — зберігає історію результатів у список
ThresholdNotifierObserver — сповіщає, якщо результат перевищує порогове значення
Принцип роботи Observer:
Кожен спостерігач підписується на подію ResultCalculated. Коли ResultPublisher викликає PublishResult(), всі підписники автоматично отримують сповіщення з результатом та назвою операції.

3. Взаємодія компонентів
У методі Main створюються:
Процесор з початковою стратегією (Square)
Видавець результатів
Три спостерігачі, які підписуються на подію

Демонструється:
Зміна стратегій (Square → Cube → SquareRoot)
Обробка набору тестових чисел
Автоматичне сповіщення всіх спостерігачів після кожної операції
Виведення повної історії операцій

4. Висновки
У ході виконання лабораторної роботи було створено гнучку систему обробки числових даних з використанням патернів Strategy та Observer.

Переваги використання Strategy:
Легке додавання нових алгоритмів без зміни існуючого коду
Можливість динамічної зміни алгоритму під час виконання програми

Переваги використання Observer:
Автоматичне сповіщення всіх залежних компонентів про зміни
Слабке зв'язування між компонентами системи
Легке додавання нових спостерігачів
Поєднання цих двох патернів дозволяє створювати розширювані системи з чітко розділеною відповідальністю між компонентами, що відповідає принципам SOLID.
