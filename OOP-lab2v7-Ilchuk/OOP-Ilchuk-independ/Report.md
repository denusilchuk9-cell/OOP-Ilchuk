IndependentWork11 — Звіт по бібліотеці Polly
Сценарій 1 — Нестабільний зовнішній API (Retry Policy)
Проблема

Зовнішній API може бути тимчасово недоступний, що призводить до HttpRequestException.

Політика Polly

Retry з експоненційною затримкою.

Обґрунтування

Більшість помилок API — тимчасові. Retry дозволяє автоматично повторити запити й дочекатися нормальної роботи сервера.

Очікувана поведінка

Перші 2 виклики API падають

Polly робить повторні спроби

Затримки: 2 сек → 4 сек → 8 сек

Спроба №3 успішна, програма продовжує роботу
Сценарій 2 — Нестабільна база даних (Circuit Breaker Policy)
Проблема
База даних інколи відповідає помилками, і якщо надсилати багато запитів поспіль, це тільки погіршує ситуацію.
Політика Polly
Circuit Breaker (автоматичне розмикання кола після кількох помилок).

Обґрунтування
Circuit Breaker дозволяє уникнути перевантаження БД.
Після декількох помилок доступ блокується на визначений час.

Очікувана поведінка

Перші 3 звернення до БД падають
Після 2 помилок Circuit Breaker відкривається
Запити блокуються на 5 секунд
Потім режим Half-Open
Якщо успішно — Circuit Breaker закривається
Сценарій 3 — Довга операція (Timeout + Fallback Policies)
Проблема
Довгі операції можуть затримувати весь застосунок.
Політика Polly
Timeout (2 секунди) + Fallback (повернути запасне значення).

Обґрунтування

Timeout зупиняє “завислі” операції.
Fallback дозволяє повернути результат, навіть якщо основна операція не завершилась.
Очікувана поведінка
Операція триває 5 секунд
Timeout спрацьовує через 2 секунди
Запускається fallback
Повертає: "Fallback result"

Загальні висновки

Polly дозволяє легко додати відмовостійкість у .NET-застосунки без складного додаткового коду.
Політики Retry, Circuit Breaker, Timeout і Fallback забезпечують стабільність і контроль над помилками.
Це робить застосунки більш надійними та безпечними при роботі з ненадійними системами.
