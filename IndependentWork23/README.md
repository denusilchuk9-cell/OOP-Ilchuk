Відповіді на контрольні питання
1. Поясніть патерн Adapter. Коли його слід використовувати?
Adapter – структурний патерн, який перетворює інтерфейс одного класу на інший, очікуваний клієнтом.

Коли використовувати:

Коли потрібно використати існуючий клас, але його інтерфейс не відповідає потребам

При інтеграції з legacy-кодом або сторонніми бібліотеками

Коли не можна змінити код адаптованого класу

2. Поясніть патерн Facade. Як він спрощує роботу зі складними підсистемами?
Facade – структурний патерн, який надає уніфікований спрощений інтерфейс до складної підсистеми.

Спрощення:

Приховує складність внутрішніх компонентів

Надає один простий метод замість багатьох викликів

Зменшує зв'язність між клієнтом і підсистемою

3. Поясніть патерн Proxy. Наведіть приклади різних типів Proxy.
Proxy – структурний патерн, який надає заступника для контролю доступу до іншого об'єкта.

Типи Proxy:

Virtual Proxy – ледаче завантаження ресурсомістких об'єктів

Protection Proxy – контроль доступу на основі прав користувача

Logging Proxy – логування викликів до об'єкта

Caching Proxy – кешування результатів операцій

Rate Limiting Proxy – обмеження кількості запитів

4. У чому ключова відмінність між Adapter, Facade та Proxy?
Патерн	Мета	Змінює інтерфейс	Додає функціональність
Adapter	Перетворення інтерфейсу	Так	Ні
Facade	Спрощення інтерфейсу	Так (новий)	Ні (тільки делегування)
Proxy	Контроль доступу	Ні (той самий)	Так (кеш, логування, контроль)
